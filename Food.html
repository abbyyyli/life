    快速加入食物

    從完整食材庫選食材，輸入吃了幾 g / ml / 顆。優格組保留為 Combo。

    分類快速加入

    同一個食材庫，依分類展開；每個食材都直接顯示成按鈕，避免下拉漏資料。

    JSON 匯入

    把 AI 輸出的 Daily Food Log JSON 貼在下方。資料庫食物用 food；偶爾吃的外食、新食物、聚餐用 custom。
    Daily Food Parser Prompt（短版）
    查看 JSON 範例
    {"date":"2026-06-06","food":[{"food":"雞里肌","gram":250},{"food":"無糖豆漿","ml":300},{"food":"大雞蛋","qty":2},{"custom":"外食雞肉便當估算","estimated":true,"nutrition":{"calories":520,"protein":32,"carbs":60,"fat":16}}]}
    使用原則：常吃食物才新增到食材資料庫；偶爾吃的外食、零食、便利商店食物，可直接用 custom JSON 記錄營養素，不需要建檔。

      食材資料庫

      大部分以每 100g / 100ml 建檔；雞蛋等天然單位可用 qty；category 可擴展，例如 drink / snack / supplement / sauce。

      新增 Combo 組合

      適合「常吃、份量固定」的組合，例如優格組、早餐組。偶爾吃的外食仍建議用 custom，不用建 Combo。
      Combo 內的 items 可使用 food/gram/ml/qty，也可以引用已存在的食材。不建議在 Combo 裡放 custom 臨時食物。

      新增食材

      你可以把營養標示丟給 AI，請它轉成 Food DB JSON，再貼進這裡匯入。
      支援單筆 object 或多筆 array。若名稱重複，會覆蓋舊資料。

      📅 History｜歷史紀錄

      所有有紀錄的日期會列在這裡。點日期可切換到該日 Dashboard。

      🧾 全部資料備份

      建議每週或每次大改前匯出一次，避免 localStorage 損壞或換裝置造成資料遺失。

      📖 Project Intro｜專案介紹

      未來開新對話時，可以先貼這段，讓 AI 知道 Food OS 的定位、資料結構與 prompt 用法。

      🧱 JSON Robustness Rules

      所有 prompt 都共用這些規則，避免 AI 輸出壞掉。

      🍱 Daily Food Parser｜每日飲食轉 JSON

      每天最常用。整天飲食、臨時多吃一樣、外食估算，都用這個 prompt。

      ➕ Food Database Generator｜新增食材

      新食物常吃超過 3 次，再用這個轉成資料庫格式。

      🌟 Combo Generator｜新增 Combo

      只有固定且常吃的組合才建立 Combo。

      設定

      Food OS v1.4.2.4.6.2 是個人紀錄工具，不是醫療或營養建議。營養標示與實際份量仍可能有誤差。
